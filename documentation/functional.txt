**FR-DLT-001: Ingest CSV Data into Bronze Layer**

* **Title**: Ingest CSV Data into Bronze Layer
* **Description**: The system must ingest CSV data from Unity Catalog Volume into the bronze layer using cloud_files.
* **Actor(s)**: Databricks Delta Tables, Unity Catalog
* **Preconditions**: CSV data is available in Unity Catalog Volume.
* **Trigger**: Triggered by the execution of the `load_customer_raw.py` and `load_orders_raw.py` scripts.
* **Main Flow / Functional Steps**:
	+ Ingest CSV data from Unity Catalog Volume using cloud_files.
	+ Enable checkpointing and incremental loads.
	+ Apply SCD Type 2 logic using id as primary key.
	+ Create columns: CreateDateTime, UpdateDateTime, IsActive.
	+ Use watermarking on order_date.
* **Alternate Flows / Exceptions**:
	+ If CSV data is missing or corrupted → Log error and notify admin.
* **Postconditions / Outputs**: Bronze layer tables (customer_raw and orders_raw) are populated with ingested data.
* **Business Rules**: SCD Type 2 logic is applied to maintain history and upsert updated data.
* **Data Requirements**:
	+ Input: CSV data from Unity Catalog Volume.
	+ Output: Bronze layer tables (customer_raw and orders_raw).
* **Assumptions**: Unity Catalog Volume is operational.
* **Dependencies**: Unity Catalog, Databricks Delta Tables.
* **Priority**: High.
* **Acceptance Criteria**:
	+ Data is ingested correctly into bronze layer tables.
	+ SCD Type 2 logic is applied correctly.

**FR-DLT-002: Transform Data into Silver Layer**

* **Title**: Transform Data into Silver Layer
* **Description**: The system must join bronze layer's customer_raw and orders_raw tables, remove nulls and duplicates, and apply SCD Type 2 upsert logic.
* **Actor(s)**: Databricks Delta Tables.
* **Preconditions**: Bronze layer tables (customer_raw and orders_raw) are populated.
* **Trigger**: Triggered by the execution of the `transform_customer_orders.py` script.
* **Main Flow / Functional Steps**:
	+ Join bronze layer's customer_raw and orders_raw tables on id.
	+ Remove nulls and duplicates using SHA256 hash-based deduplication.
	+ Apply SCD Type 2 upsert logic.
* **Alternate Flows / Exceptions**:
	+ If join fails → Log error and notify admin.
* **Postconditions / Outputs**: Silver layer table (customer_order_combined) is populated with transformed data.
* **Business Rules**: SCD Type 2 logic is applied to maintain history and upsert updated data.
* **Data Requirements**:
	+ Input: Bronze layer tables (customer_raw and orders_raw).
	+ Output: Silver layer table (customer_order_combined).
* **Assumptions**: Bronze layer tables are populated correctly.
* **Dependencies**: Databricks Delta Tables.
* **Priority**: High.
* **Acceptance Criteria**:
	+ Data is transformed correctly into silver layer table.
	+ SCD Type 2 logic is applied correctly.
