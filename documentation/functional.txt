*   **2. Title:** Combine Customer and Order Data into Silver Layer
*   **3. Description:** Load data from the `customer_raw` and `orders_raw` Bronze layer Delta tables, join them on the `id` column, cleanse the data, and load the resulting data into a Silver layer Delta table applying SCD type-2 logic.
*   **4. Actor(s):** DLT Pipeline, Auto Loader
*   **5. Preconditions:**
    *   Delta tables `bronzezone.data.customer_raw` and `bronzezone.data.orders_raw` exist and contain data.
    *   Destination Delta table `silverzone.data.customer_order_combined` either exists or can be created.
*   **6. Trigger:** New or updated data is available in the `bronzezone.data.customer_raw` or `bronzezone.data.orders_raw` tables. DLT pipeline is triggered (scheduled or continuous).
*   **7. Main Flow / Functional Steps:**
    1.  The DLT pipeline utilizes Auto Loader to load data from `bronzezone.data.customer_raw` and `bronzezone.data.orders_raw`.
    2.  The pipeline joins the two datasets on the `id` column.
    3.  Records with Null values in any column are removed.
    4.  Duplicate records are removed.
    5.  SCD type-2 logic is applied to the combined data, using `id` as the primary key.
    6.  The cleaned and combined data is loaded into the `silverzone.data.customer_order_combined` Delta table.
    7.  The following columns are automatically created and managed:
        *   `CreateDateTime`: Timestamp when the record was first inserted.
        *   `UpdateDateTime`: Timestamp when the record was last updated.
        *   `IsActive`: Boolean flag indicating if the record is the current version.
    8.  If an update to existing data is detected, the `UpdateDateTime` is updated, and the previous version's `IsActive` flag is set to `False`, while a new record is inserted with `IsActive = True`.

    *   If the source tables do not exist or are inaccessible, the pipeline should log an error and halt processing, notifying the Data Engineering Team.
    *   If the join operation fails, the pipeline should log an error and halt processing.
*   **9. Postconditions / Outputs:**
    *   Combined and cleansed Customer and Order data is loaded into the `silverzone.data.customer_order_combined` Delta table.
    *   SCD type-2 history is maintained.
    *   Null values and duplicate records are removed.
    *   `CreateDateTime`, `UpdateDateTime`, and `IsActive` columns are populated correctly.
*   **10. Business Rules:**
    *   SCD type-2 history is maintained for all changes to the combined data.
    *   Records with null values are excluded from the Silver layer.
    *   Incremental data loading processes only new or changed records on each run.
