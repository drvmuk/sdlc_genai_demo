### FR-DLT-001: Ingest CSV Data into Bronze Layer
* **Title**: Ingest CSV data from Unity Catalog Volume into Bronze Layer tables.
* **Description**: The system must ingest CSV data from Unity Catalog Volume into Bronze Layer tables using cloud_files and enable checkpointing and incremental loads.
* **Actor(s)**: Databricks, Unity Catalog
* **Preconditions**: 
  + CSV data is available in Unity Catalog Volume.
  + Unity Catalog is properly configured.
* **Trigger**: Manual trigger or scheduled job.
* **Main Flow / Functional Steps**:
  1. Read CSV data from Unity Catalog Volume using cloud_files.
  2. Enable checkpointing and incremental loads.
  3. Apply SCD Type 2 logic using id as primary key.
  4. Include columns: CreateDateTime, UpdateDateTime, IsActive.
  5. Use watermarking on order_date.
  6. Write data to bronzezone.data.customer_raw and bronzezone.data.orders_raw tables.
* **Alternate Flows / Exceptions**:
  + If CSV data is missing or corrupted, log error and notify admin.
  + If Unity Catalog is not available, retry connection.
* **Postconditions / Outputs**: Data is ingested into Bronze Layer tables.
* **Business Rules**: 
  + Use id as primary key for SCD Type 2 logic.
  + Include CreateDateTime, UpdateDateTime, IsActive columns.
* **Data Requirements**:
  + Input: CSV data from Unity Catalog Volume.
  + Output: bronzezone.data.customer_raw and bronzezone.data.orders_raw tables.
* **Assumptions**: Unity Catalog is properly configured and accessible.
* **Dependencies**: Unity Catalog, Databricks.
* **Priority**: High.
* **Acceptance Criteria**:
  + Data is correctly ingested into Bronze Layer tables.
  + SCD Type 2 logic is applied correctly.
  + Watermarking is applied on order_date.

### FR-DLT-002: Transform Data in Silver Layer
* **Title**: Join Bronze Layer tables and transform data into Silver Layer.
* **Description**: The system must join Bronze Layer's customer_raw and orders_raw tables, remove nulls and duplicates, and apply SCD Type 2 upsert logic.
* **Actor(s)**: Databricks.
* **Preconditions**: 
  + Data is available in Bronze Layer tables.
* **Trigger**: Manual trigger or scheduled job.
* **Main Flow / Functional Steps**:
  1. Join bronzezone.data.customer_raw and bronzezone.data.orders_raw tables on id.
  2. Remove nulls and duplicates using SHA256 hash-based deduplication.
  3. Apply SCD Type 2 upsert logic.
  4. Write data to silverzone.data.customer_order_combined table.
* **Alternate Flows / Exceptions**:
  + If data is missing or corrupted, log error and notify admin.
* **Postconditions / Outputs**: Transformed data is available in Silver Layer table.
* **Business Rules**: 
  + Use SHA256 hash-based deduplication.
  + Apply SCD Type 2 upsert logic.
* **Data Requirements**:
  + Input: bronzezone.data.customer_raw and bronzezone.data.orders_raw tables.
  + Output: silverzone.data.customer_order_combined table.
* **Assumptions**: Data is correctly ingested into Bronze Layer tables.
* **Dependencies**: Databricks, Bronze Layer tables.
* **Priority**: High.
* **Acceptance Criteria**:
  + Data is correctly transformed and loaded into Silver Layer table.
  + SCD Type 2 upsert logic is applied correctly.
